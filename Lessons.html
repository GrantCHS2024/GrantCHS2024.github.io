Event Listeners

Just like jQuery's '.on()' event function... 'addEventListener()' does the same without jQuery!
Check out this button:

I used .addEventListener("click", () => {}) to make it work!
The first part is the event type, in this case "click".
The second part is an arrow function, which is a shorter way to write a function.
Inside the arrow function is what happens when the event is triggered.
Take a look at this function:

document.addEventListener("keydown", (event) => {
if(event.key === "c"){
alert("Boo! Did I scare you? You pressed the c key");
}
});

That's right! It listens for if a key is pressed down, then the 'event' helps it check if the key is "c"!
If it is, then it alerts you!

Canvas

Canvas provides a blank area on a web page where graphics can be drawn dynamically using JavaScript.
To access it, you use 'canvas' tags. You can give it a class or id and bring it to JavaScript.
But by doing that, to access the drawing features, you'll need to establish it's context.
You can do this by naming a new variable and setting it to '(canvas variable name).getContext(2d)'
Now you can use that new variable to DRAW inside of that rectangle.

The 'event' in the parameter is a special object passed to an event handler function when an event occurs.
For this one, it mainly tracks the mouse's position and sets the x and y values to its variables.

Another thing you can add for making games inside canvas is 'ctx.clearRect(0, 0, canvas.width, canvas.height)' at the VERY top of your draw function.
What this does is wipes the canvas so it can create a new frame instead of leaving the previous frame on screen.
Using that you can add a separate function that updates all values you have and calls the draw function(s) so they'll be drawn accordingly... Then, you can create cool games on your own!
